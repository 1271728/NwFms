# 课题阶段汇报（可直接粘贴到系统）

## 一、前期工作小结
前期围绕“高校科研经费报销管理系统”完成了从技术选型、数据库设计到核心业务开发的闭环推进。技术方案方面，后端采用 Spring Boot + MyBatis 架构，结合 Sa-Token 完成登录鉴权与基于角色的访问控制；前端采用 Vue 3 + TypeScript + Vite，形成前后端分离实现路径。当前系统已建立较完整的业务模块骨架并进入联调阶段：
1. **组织与权限基础能力**：已实现认证鉴权、角色菜单权限控制、用户管理与组织机构相关能力，支持管理员、二级单位管理员、财务人员、项目负责人等多角色使用。
2. **项目管理主线**：已完成项目立项、成员管理、状态流转与审批日志相关实现，具备项目从草稿到审批的全流程支撑。
3. **预算与调整能力**：已实现预算科目树与项目预算总览，并支持预算调整申请及流程化审批，满足预算动态管理需求。
4. **报销与审批能力**：已实现报销单创建、明细维护、附件上传、提交审核及审批记录；并在流程中心统一处理待办/已办，提高业务协同效率。
5. **配套与统计能力**：已实现消息中心、支付管理、档案归档、报表统计与首页看板等功能模块，能够对核心业务形成管理闭环。

总体上，课题当前采用“统一认证 + 角色驱动 + 工作流贯通 + 报表支撑”的技术路线，阶段性目标基本达成，系统已具备可演示、可验证的核心业务能力。

## 二、存在的问题及下一步工作计划
### （一）当前存在的问题
1. **自动化测试覆盖不足**：当前测试主要为基础启动测试，针对项目、预算、报销、审批等关键流程的单元测试与集成测试覆盖还不充分。
2. **系统健壮性仍需增强**：在高并发、异常输入、边界金额校验、跨角色并发审批等场景下，仍需进一步做稳定性与一致性验证。
3. **运维与发布规范待完善**：目前以开发环境联调为主，生产部署方案、配置分层、日志审计与备份恢复策略还需系统化补齐。
4. **用户体验可继续优化**：复杂表单交互、审批意见提示、列表检索效率和消息触达及时性仍有优化空间。

### （二）下一步工作计划
1. **补齐测试体系（短期优先）**：围绕项目立项、预算调整、报销审批等主链路补充单元测试与接口集成测试，形成可回归的测试清单。
2. **完善流程与规则校验**：进一步细化审批节点规则、金额与预算联动校验、异常回滚与日志追踪机制，提升流程可靠性。
3. **推进部署与运维建设**：完成环境分层配置、数据库迁移脚本规范、运行监控与告警方案，提升系统可维护性与可用性。
4. **优化前端交互与性能**：聚焦高频页面（项目、预算、报销、审批中心）进行表单体验、查询性能和可视化展示优化。
5. **准备结题材料**：同步沉淀需求说明、总体设计、数据库设计、接口文档、测试报告与演示材料，确保课题成果可验收、可复用。
